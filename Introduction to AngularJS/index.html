The Why and What of AngularJS

  • Front-end JavaScript framework for creating web applications

    You can basically structure your Backend like you wish.

    • Good for dynamic web sites / web apps (CRUD based)
    • Framework imposes a structure that is good for organization
    • Helps create responsive (meaning fast) websites
    • Easy to test - to create software that is easily maintained
  • Open source maintained by Google
  • MVC pattern
  • Handles common (and often trying tasks) such as DOM manipulation, UI updating based on dta or input, registering callbacks.

  • Declarative programming

                        
                        <body ng-app="spicyApp2">
                            <div ng-controller="SpicyController">
                                <input type="text" ng-model="customSpice">
                                <button ng-click="spicy('chili')">Chili</button>
                                <button ng-click="spicy(customSpice)">Custom spice</button>
                                <p>The food is{{spice}} spicy</p>
                            </div>
                        </body>

    The intention of the application is expressed or declared in the HTML for Angular. Instead of JQuery which is only visible in the js Script, so that you don't any idea how the site behaves.

How AngularJS works

  • AngularJS will initialize when the DOM content is loaded.

    Angular uses dependancy injection to make modules more testable and reuseable.

  • Looks for the ng-app directive - if its found, that is the root of the app
  • Directives can be declared in a varity of ways: typically with the ng- prefix, but you can use data-ng in case you have to support older Browsers.
  • It will laod the module associated with the directive if specified.

What are AngularJS Modules

  • Container for the various parts of your application(controllers, services etc) the building blocks of your app.
  • Declaritive - easy to understand
  • Maintainable, readable, testable
  • Define dependencies for our app
  • Module API

Creating an AngularJS Module

  • A Module is comprised of configuartion and run blocks
  • Configuration blocks(providers) - executed during configuration and registration. Only providers and constants can be passed. Is doing the setup work
  • Run blocks - happen after the injector is created. Only instances and constants can be passed in. runs the actual code.
  • Some convienience methods for this
  • Run blocks is like a main methode - it kickstarts the application
  • Modules can depend on other modules
  • They are only loaded once.
                angular.module('myModule', []).config(function(injectables){
                    //This is an example of config blocks.
                    //You can have as many of these as you want.
                    //You can only inject Providers (not instances)
                    //into config blocks
                }).run(function(injectables){
                    //This is an example of a run block
                    //You can have as many of these as you want.
                    //You can only inject Providers (not instances)
                    //into config blocks
                });
