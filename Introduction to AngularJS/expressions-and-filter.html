Whats this Chapter about

What is an Expression

  • JavaScript code snippets placed in bindings {{Hello World}}
  • Evaluated against a scope object
  • Forgiving, don't throw an error
  • Can use filters with expressions to format data
  • Expressions do not have access to window, socument or location
  • Angular Expression Doku

What can you do in an Expression

It is possible to do a onetime bind

For Elements which get bind one time and than never change.

Think of a header on a page. That's not going to change. It's always going to be the same header but maybe you need to dynamically populate it for whatever reason. So maybe it's coming from the NAV and the NAV has something associated with it. You got... whatever it may be. You know it's not going to change. You maybe just want to bind that once so that when that whole module runs, gets configured and then runs and that link is made, then it just does it once and it's not worried about watching that item all the time.

Using Expressions

What are Filters

Basically filters format the value of an expression is the easiest way to think of them.

You can use it in the view but you can use them in controllers and services and so a prime example might be you have a service that goes out and it gets a data feed and when it comes back you're going to want to sort that data feed on a certain property before it gets populated to the view.

  • Formats the value of an expression
  • Filters can be applied in the view {{employees.array | filtername: '...'
  • Filters applied in the view can be costly as they are re-evaluated.
  • Filters can be used elsewhere: controllers, sevices etc.

Buildin Filters

  • orderBy
  • limitTo
  • lowercase and uppercase
  • filter
  • currency
  • number
  • date

Example:
<div ng-repeat="item in event.events | orderBy: 'date' | limitTo: '2'" ..... >

The Example gets all elements in the event array and order it by 'date' and limit the output to two elements
