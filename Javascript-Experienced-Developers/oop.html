Object Oriented Programming in JavaScript

Though many people are unaware, Javascript is a great OOP language. Unlike others like C# and Java, JavaScript supports inheritance through prototyping.

Protoyping is simply first creating the behavior of any constructor and then reusing it to create other Objects.

In JavaScript, there is no real class statement. Insead a constructor is made as a function.

JavaScript does have some built in objects. But the object at the top, top, top of the hierarchy would be the object object. So all objects that are made, whether built in or customized, which are the ones that we would be making. They all pretty much inherit all the functionalities of the object object.

Creating Objects

How to Create An Object

  • Using Object Literals

  • With this method, you simply initialize all the properties of the object within brackets
  • The properties are separated by commas
    •                 
                  var collegeAlgebra = {
                    level: "freshman",
                    difficulty: "hard",
                    expectedGrade: 98,

                    textbook: function(){console.log("Here comes the Mouse");}
                  }

  • Using the Object Constructor

  • With this method, you insantiate the Object by calling the Object constructor
  • Properties are added using dot notation
    •                 
                  var collegeAlg = new Object;
            
                  collegeAlg.level = "freshman";

                  collegeAlg.textbook = function(){
                    console.log("Here comes the Mouse");
                  }

Accessing Object Properties

Dot Notation: Using a period between the object name and property

Bracket Notation: Using a bracket between the object name and property
Also lets you access property/methode name through other variables

            
                var collegeAlg = new Object;
            
                  collegeAlg["level"] = "freshman";

                  collegeAlg["textbook"] = function(){
                    console.log("Here comes the Mouse");
                  }

And so the bracket notation gives you the ability to be a bit more dynamic there. So if maybe at runtime you're gonna figure out what property it is that you wanna display, you could then use the bracket notation there.

            
              var propName = 'level';
              var methName = 'textbook';

              collegeAlg[propName]; // To Assign the Value;
              collegeAlg[methName]; // To Assign a function;

              collegeAlg[methName](); // To Assign and Call a function;

Deleting a property

Deleting a property from an object is something as simple using the delete operator. You cannot delete properties that were inherited.

            
              var collageAlg = {
                level: "freshman",
                difficulty: "hard",
                expectedGrade: 98
              }

              delete collageAlg.level;

Constructors and the Prototype Object

Constructors

  • Just like other OOP languages, constructors are used to set an object's properties and methods.
  • constructors are invoked the moment an object is instantiated
  • In Javascript, constructors are written in the form of a function using the function keyword
            
              function Fruit(name, color){
                console.log("Object created by constructor");

                this.name = name;
                this.color = color;

                this.type = function(){
                  console.log("Hi, I am a fruit");
                }
              }

              var orange = new Fruit("orange", "orange");
              orange.type;
              console.log(orange);
