Modules

  • Modules in JavaScript are small units of reusable code
  • Mudules are used to export a value, rather than to define a type
  • Usually export objects, functions or constructors
  • Modules usually belong to a package, which is made of many different modules.

What is RequireJS than?

So, RequireJS is actually a framework that is able to load different files, and modules onto your JavaScript code. What you'll be using is a Require keyword.

  • A framework used to load Javascript files and modules
  • Uses the Asynchronous Modul Definition to load files
    - Each module starts loading through asynchronous requests in a given order.
  • Can be used with jQuery, Node.js and Dojo

Two Important Key functions

  • require() require() is used to run immediate functionalities.
    Example

                    var messages = require('./pathToModule');
                console.log(messages.getHello());

  • define() define() is used to define modules for use in multiple locations.
    Example:

                    define(function() {
                  getHello: function(){
                    return 'Hello World';
                  }
                });

What is MVVM

Now the next big thing that comes up quite frequently in the JavaScript world, is these four little letters, MVVM. Or Model-View-ViewModel.

  • Model-View-ViewModel
  • A framework for automatically updating a UI
  • Several are availiable
    -Knockout
    -Angular
    -Ember
    -Backbone

well what does it do for you? Well in a nutshell, it will automatically update your UI. So if you have a JavaScript object in memory, and something changes. So maybe the user did something. Maybe with a web socket or a signal R. You got a signal from somewhere else. And Something changed inside of your JavaScript and you wanna be able to automatically update the UI, MVVM can do that.
